# 📏 发展规划

## 第零阶段：保障 CoCo 功能

对应里程碑：https://gitee.com/oldsquaw-coco-next/CoCo-Next/milestones/221656

保证 CoCo 功能能够正常使用，使用户可以从 CoCo 无缝迁移。

## 第一阶段：补全基础功能

对应里程碑：https://gitee.com/oldsquaw-coco-next/CoCo-Next/milestones/221607

在该阶段，我们会致力于 CoCo 的一些基础问题，如：Undefined 修复、移除自定义控件。这些功能虽然大多已有其他解决方案，但是用起来太麻烦了。同时，这也是社区工具的基础功能。

这一阶段计划实现的内容：

- 功能
  - 扩展性
    - 自定义控件
      - 限制
        - [x] 移除导入自定义控件和运行作品时的对自定义控件的关键词检查
        - [x] 移除对自定义控件的全局变量访问限制
        - [x] 移除对使用第三方自定义控件的作品的协作限制
        - [ ] 移除对使用第三方自定义控件的作品的分享限制（绕审核）
  - 积木
    - [x] 临时列表积木
  - 屏幕
    - [x] 移除屏幕数量上限
- 体验
  - [x] 导入自定义控件出错时提示
  - 移动端优化
    - 触屏优化
      - [x] 拖拽
      - [ ] 点击
    - [ ] 小屏优化
  - [x] 移除自定义控件
  - [x] 自动更新控件商城控件的链接
- 问题修复
  - [x] Undefined 修复

## 第二阶段：提升可扩展性

可扩展性是提升编辑器用途广泛程度、社区活跃度的关键。我们计划通过自定义控件和插件系统来提升可扩展性。

这一阶段计划实现的内容：

- 扩展性
  - 自定义控件
    - 特性
      - [ ] 静态方法
      - [ ] 可变参数
      - [ ] 字典/列表类型的默认值
      - [ ] 高阶函数
      - [ ] 自定义属性编辑器
      - [ ] ES Module
      - [ ] 第三方开发工具
      - [ ] 外部依赖
  - [ ] 插件
